# reality-check-ai

RealityCheck AI is a Chrome browser extension that detects AI-generated images in real time while you browse the web.

Instead of requiring users to upload images or manually check suspicious content, the extension runs silently in the background. The extension captures screenshots when new content appears or when the page settles during browsing, sends the screenshot to a detection backend, and analyzes the image using the **Sightengine AI detection API**. The result is returned instantly and displayed as a floating badge directly on the webpage, showing the probability that the content being viewed was AI-generated.

The goal is simple: provide passive protection directly inside the browsing experience, where misleading AI-generated media actually appears.

---

# How It Works

1. The user browses normally in Chrome  
2. The extension’s background service worker captures the visible tab when new content loads  
3. The screenshot is sent via POST request to a FastAPI backend  
4. The backend sends the image to Sightengine’s AI detection model  
5. Sightengine analyzes the image and returns a probability score  
6. The result is sent back to the extension  
7. A floating badge appears on the page showing whether the image is likely AI-generated  

This architecture allows detection to happen continuously and automatically, without any action from the user.

# Technology Stack

RealityCheck AI combines several technologies across the browser, backend, and cloud infrastructure to create a real-time detection pipeline.

---

## Chrome Extension (Client Layer)

The Chrome extension is the user-facing component of the system.

### Manifest V3

We built the extension using Chrome Extension Manifest V3, the current standard for Chrome extensions. It defines the permissions, scripts, and resources needed for the extension to run securely within the browser.

### background.js

The background service worker acts as the engine of the extension. It captures the active browser tab when page changes occur and sends screenshots to the backend for analysis.

### content.js

This script runs on every webpage and is responsible for injecting the floating overlay badge into the page. It listens for detection results and updates the badge dynamically.

### overlay.css

The overlay styling ensures the badge appears consistently across all websites, positioned unobtrusively in the corner of the screen.

Together these components allow the extension to monitor content automatically without disrupting the user’s browsing experience.

---

# Backend (Detection Layer)

The backend acts as the bridge between the browser extension and the AI detection model.

### Python

**Python** was chosen because of its strong ecosystem for AI and machine learning tools, as well as its compatibility with modern web frameworks.

### FastAPI

We implemented the backend using FastAPI, a high-performance Python web framework designed for building APIs quickly and efficiently.

FastAPI handles incoming requests from the extension and forwards the image data to the detection service.

### Uvicorn

The FastAPI application runs on Uvicorn, an ASGI server that allows the backend to handle asynchronous requests efficiently.

---

# AI Detection Model

## Sightengine API

The system currently uses the Sightengine AI-generated image detection API.

Sightengine analyzes images and returns a probability score indicating whether the image was generated by artificial intelligence.

We chose Sightengine for several reasons:

- It provides a ready-to-use detection model, allowing rapid integration during the hackathon  
- The API returns clear probability scores that can be easily displayed in the extension interface  
- It removes the need to train and maintain our own model within the limited hackathon timeframe  

Using an external detection API allowed us to focus on building the real-time browser detection pipeline, which is the core innovation of the project.

At the moment, the application relies on this API to perform all detection, which means the extension requires the backend service to be running in order to function.

# Cloud Infrastructure

## AWS EC2

The FastAPI backend is hosted on an AWS EC2 instance, which allows the extension to communicate with a publicly accessible API endpoint.

EC2 was selected because it provides:

- quick deployment  
- reliable uptime  
- scalable infrastructure if the project grows beyond the hackathon prototype  

This setup ensures the detection service can be accessed from any browser running the extension.

# How Everything Works Together

RealityCheck AI operates as a continuous pipeline:

**Chrome Extension → FastAPI Backend → Sightengine Detection → Browser Overlay**

1. The extension captures what the user is seeing  
2. The backend processes the request  
3. Sightengine analyzes the image  
4. The result is returned and displayed instantly in the browser  

By combining browser automation, cloud infrastructure, and AI detection services, the system creates a fully automated real-time deepfake detection experience directly inside the web browsing environment.

# Why This Approach Matters

Most existing AI-image detection tools are reactive. They require the user to already suspect that something might be fake, save the image, visit a detection website, upload the file, and wait for a result. In practice, this rarely happens. The people most vulnerable to misinformation or scams are the least likely to take those extra steps.

RealityCheck AI flips this model.

Instead of waiting for suspicion, detection happens automatically at the moment the user encounters the content. By embedding the detector directly inside the browser, the system analyzes images as they appear on screen and provides immediate feedback through a lightweight visual indicator.

This approach shifts AI detection from a manual verification tool to a passive safety layer, protecting users in the environments where misleading content actually spreads: social media feeds, online advertisements, and news websites.

The result is a system that does not rely on technical expertise, suspicion, or extra effort from the user. It simply runs in the background, helping people make more informed judgments about what they see online.
